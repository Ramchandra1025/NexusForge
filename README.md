# NexusForge 🛠️  
### From 2nd-Year CSE Students to Building a Visual IDE

Over the past few months, my team and I have been diving deep into how professional design tools like Figma and Webflow actually work under the hood. That exploration led us to build **NexusForge**—a visual editor crafted from scratch.

This project became a major learning experience in **state management, DOM manipulation, and responsive system design**. We intentionally avoided heavy frameworks to better understand performance, architecture, and the fundamentals of the web.

---

## 🚀 What we built into NexusForge:

- **Dynamic Breakpoints**: Real-time canvas scaling for Desktop, Tablet, and Mobile views  
- **Command Palette**: A keyboard-first workflow (Ctrl + K) for faster navigation  
- **Component Foundry**: A drag-and-drop system for building UI elements like Typography, Buttons, and Forms  
- **Dual-Engine UI**: Seamless Dark and Light modes powered by custom design tokens  

---

Building this pushed our understanding of **vanilla JavaScript, modern CSS (Grid & Flexbox), and scalable UI architecture** much further than any tutorial could.

It’s one thing to use tools—but it’s a completely different experience to build one from the ground up.

This was a true team effort, and I’m proud of what we were able to create together.
